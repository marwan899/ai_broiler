# منصة إدارة حقول فروج اللحم — Eng Marwan ALnasree

هذا المشروع يوفر تطبيق ويب كامل باللغة العربية لإدارة حقول فروج اللحم (Ross 308) مع واجهة للمربين ولوحة تحكم مستقلة للإدارة.

## المتطلبات
- Python 3 (لتشغيل خادم المعاينة البسيط).
- أي متصفح حديث يدعم اللغة العربية واتجاه الكتابة من اليمين إلى اليسار.

## تشغيل التطبيق محليًا
1. من داخل مجلد المشروع نفّذ:
   ```bash
   ./serve.sh
   ```
   يمكن تحديد منفذ مخصص عبر تمرير الرقم مثل `./serve.sh 9000`.
2. افتح المتصفح على عنوان واجهة المربين: `http://localhost:8000/index.html` (أو المنفذ الذي اخترته).
3. للوصول إلى لوحة الإدارة المستقلة استخدم `http://localhost:8000/admin.html`.

> **تلميح:** يظهر السكربت روابط الواجهتين عند التشغيل لتذكيرك بالمسارات الصحيحة.

## خطوات التجربة الأولى
1. **تسجيل مربي جديد:** من `index.html` أدخل الاسم الكامل، رقم الهاتف، كلمة المرور، وعدد الصوص المستلمة. سيتم حفظ الطلب في التخزين المحلي بانتظار الموافقة.
2. **تسجيل دخول الإدارة:** من `admin.html` استخدم بيانات الدخول الافتراضية:
   - اسم المستخدم: `admin`
   - كلمة المرور: `ross308`
3. **الموافقة على المربي:** بعد الدخول، وافق على طلب التسجيل ثم فعّل حسابه. يمكن أيضًا تعديل بياناته أو تعطيله أو حذف سجلاته.
4. **دخول المربي المعتمد:** عد إلى واجهة المربين وسجّل الدخول باستخدام رقم الهاتف وكلمة المرور اللذين أدخلهما أثناء التسجيل. لن يسمح التطبيق بالدخول قبل الموافقة من الإدارة.
5. **إرسال تقرير يومي:** بعد الدخول، أدخل بيانات العمر، عدد الطيور الحية، متوسط الوزن بالغم، كميات العلف والماء، وسجلات اللقاحات والملاحظات. يتم تحديث مؤشرات Ross 308 (الوزن المستهدف، برنامج الإضاءة، درجة الحرارة، واستهلاك الماء) تلقائيًا بناءً على العمر.

## إدارة الدورات والسجلات
- لا يمكن تعديل الاسم أو عدد الصوص بعد الموافقة حتى تبدأ دورة جديدة عبر لوحة الإدارة.
- يمكن للإدارة تعطيل مربي، إعادة ضبط سجلاته، أو حذف حسابه بالكامل.
- التخزين يتم محليًا على المتصفح (LocalStorage). لإعادة التهيئة اليدوية يمكنك مسح بيانات الموقع من إعدادات المتصفح.

## نشر المشروع على GitHub
1. **تجهيز المستودع البعيد:** أنشئ مستودعًا جديدًا على GitHub (عام أو خاص) بدون ملفات افتراضية.
2. **إعداد المستودع المحلي:** من داخل هذا المجلد تأكد أن جميع التعديلات جاهزة، ثم نفّذ:
   ```bash
   git status
   ```
   للتحقق من حالة الملفات.
3. **تسمية الفرع الرئيسي (اختياري):** إذا كنت تفضل استخدام `main` بدل `master` فقم بتغييره:
   ```bash
   git branch -M main
   ```
4. **إضافة المستودع البعيد:** استبدل `YOUR-REPO.git` بعنوان HTTPS أو SSH الخاص بمستودعك:
   ```bash
   git remote add origin https://github.com/USERNAME/YOUR-REPO.git
   ```
5. **رفع الكود إلى GitHub:** ادفع الفرع الحالي إلى GitHub:
   ```bash
   git push -u origin main
   ```
   بعد نجاح العملية ستجد الملفات (`index.html`, `admin.html`, `serve.sh`, `README.txt`) في المستودع البعيد، ويمكنك مشاركة الرابط مع فريقك أو العملاء.

## نشر عالمي عبر GitHub Pages
يأتي المستودع مجهزًا بخط تجميع آلي يعتمد GitHub Pages كي يصبح التطبيق متاحًا للعالم من دون إعدادات إضافية.

1. بعد رفع الملفات إلى الفرع `main` على GitHub، انتقل إلى إعدادات المستودع > **Pages** وتأكد أن المصدر مضبوط على "GitHub Actions" (يتم ذلك تلقائيًا عند أول تشغيل).
2. سيعمل سير العمل `.github/workflows/deploy.yml` عند كل دفع إلى `main` أو عند تشغيله يدويًا من تبويب **Actions**.
3. يقوم السير بإنشاء مجلد `site/` يحتوي على `index.html`, `admin.html`, و`README.txt` ثم يرفعه إلى GitHub Pages.
4. بعد اكتمال النشر ستجد الرابط العمومي في صفحة الإصدارات الخاصة بالبيئة `github-pages`. انسخ الرابط وشاركه مع المربين والإدارة ليصلوا للتطبيق من أي مكان في العالم.
5. في حال رغبت بتخصيص نطاق مخصص أضف ملف `CNAME` داخل مجلد المشروع قبل الدفع ثم حدّث إعدادات النطاق من لوحة GitHub Pages.

> **معلومة:** أي تغييرات مستقبلية على ملفات الواجهة ستُعاد نشرها تلقائيًا خلال دقائق من دفعها إلى الفرع الرئيسي.
